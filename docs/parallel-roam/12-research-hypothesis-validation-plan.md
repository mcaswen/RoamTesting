# 研究假设与验证计划（初稿 v0.2）

> 暂定题目：面向固定拓扑预算的 GPU 自适应网格细分：兼容闭包感知的全局资源分配
> 日期：2026-07-13；状态：用于决定是否正式进入论文实现阶段

## 研究问题

### CBT 已经完成的工作

并发二叉树（CBT）首先将动态二叉树编码为紧凑位域，并配合求和归约树完成活动叶节点计数、索引和间接任务生成，使大量线程能够通过位操作并发执行 split/merge，而不依赖 CPU 指针树和串行优先队列。CBT 2024 将这一结构扩展到一般多边形网格：每条半边对应一个二分器，显式维护二分器数据和邻接关系，并把 CBT 作为 GPU 上可并发分配、回收和压缩的拓扑内存池。

其更新流水线大致包括候选分类、split、内存预留、二分、兼容关系传播、简化、索引重建和验证。需要分裂一个网格单元时，算法沿邻接关系确定兼容链，并传播必要的强制分裂，使最终网格保持拓扑相容；需要简化时，则检查合法的合并结构并传播更新。由此，CBT 主要解决了以下问题：

- 动态拓扑如何长期驻留在 GPU，而不是每帧由 CPU 重建并上传；
- 大量二分器如何并发分配、回收、索引和生成间接工作负载；
- split/merge 的依赖如何传播，从而避免裂缝和非法邻接；
- 不规则动态拓扑如何保持足够紧凑，并达到实时更新性能。

### CBT 尚未解决的预算分配问题

CBT 2024 的 `GenerateCommands`（命令生成）和分类阶段主要依据投影面积阈值判断细分、简化或保持不变。细分候选通过原子计数写入命令列表；执行 split 时，每个候选按兼容链的保守最大需求原子扣减剩余内存，空间不足便回滚计数并放弃该候选。这个机制保证了**不会越界写入内存池**，但获批顺序仍取决于 GPU 的实际线程调度和原子竞争顺序，而不是候选的全局质量收益。

因此，当候选需求超过显存池容量时，现有方法仍有五个缺口：

1. **没有全局优先级。** 低收益候选可能先占用空间，使更重要的候选随后失败，产生优先级倒置。
2. **没有收益/成本联合决策。** 兼容链长度不同，一个昂贵 split 可能挤掉多个总体收益更高的廉价 split。
3. **没有闭包重叠计费。** 多个候选可能共享强制分裂操作；逐候选保守预留会重复估计成本并留下未使用空间。
4. **没有饱和状态下的资源再分配目标。** 即使新视点出现高误差区域，已经被低收益旧细节占用的二分器也不会自动按全局价值重新交换。
5. **阈值不等于预算优化。** 调整投影面积阈值只能改变候选数量，不能严格保证固定预算被最有价值的区域使用，也不能直接控制最终误差或预算利用率。

### 本文拟研究的问题

本研究拟回答：**在固定二分器容量和实时 GPU 时间约束下，能否考虑兼容闭包的依赖、重叠和实际成本，对 split/merge 进行全局预算感知分配，从而在保持无裂缝拓扑的同时降低视觉误差？**

具体分解为四个问题：

- **问题 1：质量收益。** 如何用可并行计算且可解释的指标估计一次 split 带来的屏幕空间几何误差下降，而不是只判断当前三角形是否超过面积阈值？
- **问题 2：依赖成本。** 如何在 GPU 上计算或近似候选的兼容闭包，并对多个闭包的共享操作去重，从而得到严格可行的真实预算成本？
- **问题 3：全局选择与交换。** 如何在容量约束下选择高收益候选，并通过低损失 merge 与高收益 split 的交换，使预算能够随视点和地形特征重新分布？
- **问题 4：并行性与保证。** 如何避免串行全局优先队列，同时保证不超预算、闭包完整、拓扑正确，并把额外调度开销控制在实时范围内？

形式上，候选 `i` 具有预测收益 `g_i` 和兼容闭包 `P_i`；选择集合 `S` 的真实成本是 `|⋃_{i∈S} P_i|`，不是各候选成本之和。研究目标不是声称求得严格全局最优，而是得到**严格不超预算、依赖感知且适合 GPU 并行的近似调度方法**。

## 可证伪假设

- **H1 质量：**相同活动三角形数或二分器容量下，相比 CBT 2024 的先到先分配策略，本文方法能稳定降低最大值和 P95 屏幕空间误差。
- **H2 预算：**共享闭包去重和真实成本统计能减少保守预留造成的空闲，且内存超额量始终为零。
- **H3 调度：**稳定优先级和确定性分配能减少 GPU 执行顺序导致的优先级倒置，并保持拓扑正确。
- **H4 性能：**候选评分、闭包分析、选择和提交的额外 GPU 开销不会抵消质量收益。
- **H5 动态性：**池饱和时，低损失 merge 与高收益 split 的预算交换比仅处理新增 split 更能适应视点移动，并可通过滞后阈值和冷却期控制抖动。

## 方法方向与基线

候选方案是在冻结的单轮拓扑上计算屏幕空间误差收益和规范化兼容闭包，再使用收益/成本优先级、GPU 基数排序或分桶选择、闭包去重、确定性前缀分配和剩余预算填充。初期只验证细分，H1-H4 成立后再加入 merge/split 交换；具体算法在前置可行性实验完成前不冻结。

| 方法 | 角色 |
|---|---|
| CBT 2024 先到先分配策略 | 同 GPU 拓扑下的主要直接基线 |
| GPU 误差 Top-K | 判断普通排序是否已经足够的消融基线 |
| 本文依赖感知调度器 | 待验证方法 |
| 项目现有经典 CPU ROAM | 固定三角形预算下的优先级质量参考 |
| 项目现有数据导向 CPU ROAM | 数据导向 CPU 吞吐基线 |
| 小规模穷举、整数线性规划或串行精确贪心 | 估计与最优参考的差距 |

## 验证与决策标准

所有方法使用相同高度图、相机路径、最大深度、误差评估器和最终三角形预算。场景覆盖平坦/高曲率混合、山脊峡谷、长兼容链、遮挡后显露和不少于三类真实数字高程模型（DEM）；重点测试候选需求为容量 2 倍和 4 倍的超额需求场景。记录最大/P95/平均屏幕空间误差、深度与法线误差、预算利用率、优先级倒置、各 GPU 阶段时间、拓扑变更量、裂缝、非法邻接和结果确定性。

**继续条件：**多个真实场景中，本文方法相较先到先分配策略对最大值或 P95 误差稳定改善约 10%，内存超额量为零，预算利用率不退化，选择阶段在参考 GPU 上不超过约 1 毫秒或拓扑更新时间预算的 20%。**停止或转向条件：**改善不足 5%、只存在于人工反例、简单阈值或 Top-K 已取得相近质量，或调度成本长期高于收益。
